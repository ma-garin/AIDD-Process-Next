# sqec.md — AIDD Process Next 受入条件・品質基準

## 1. 対象

本書は、AIDD Process Next v0.1 の受入条件、品質ゲート、テスト観点を定義します。

対象アプリは `docs/index.html` を入口とする静的Webアプリです。v0.1ではバックエンド、認証、DB、AI APIを持ちません。

---

## 2. 品質ゲート

| ID | ゲート | 必須条件 |
|---|---|---|
| QG-1 | 日本語UI | 主要画面、設問、レポート、自己点検プロンプトが日本語で表示される |
| QG-2 | B2Bデザイン | 日本の業務ツールとして違和感がない低彩度・高可読性のデザインである |
| QG-3 | データ分離 | 設問は `docs/data/questions.js` に集約され、UIへ直書きしない |
| QG-4 | スコア再現性 | 同じ回答から同じスコア・成熟度・リスクが算出される |
| QG-5 | リスク表示 | スコア0〜1の項目が優先リスクとして表示される |
| QG-6 | 改善Issue | 優先リスクから改善Issue案が生成される |
| QG-7 | レポート | Markdownレポートを表示・コピーできる |
| QG-8 | 自己点検 | 実装前・PR前・マージ前のAI自己点検プロンプトを表示・コピーできる |
| QG-9 | セキュリティ | 入力値はHTMLエスケープされ、外部送信されない |
| QG-10 | リポジトリ衛生 | `.git`、`.DS_Store`、`.playwright-mcp/`、`__MACOSX/` を含めない |

## 4. スコアリング仕様

- 各設問は0〜3点
- カテゴリスコア = 回答済み点数合計 ÷（回答済み設問数 × 3）× 100
- 総合スコア = カテゴリスコアの加重平均
- v0.1のカテゴリ重みは均等10%
- N/Aは母数から除外

| 総合スコア | レベル |
|---:|---|
| 0〜24 | Lv.0 未統制 |
| 25〜49 | Lv.1 管理開始 |
| 50〜74 | Lv.2 仕組み化 |
| 75〜100 | Lv.3 最適化 |

---

## 5. UX・デザイン基準

- 日本語の読みやすさを優先する
- 派手なAI風デザインではなく、QA/PM/管理職が業務で開ける画面にする
- 色だけで状態を伝えず、ラベル・アイコン・文言を併用する
- 重要情報の優先順位は「総合スコア → 成熟度 → 優先リスク → 改善Issue」
- レポート画面は、QAリード、PM、管理職へそのまま共有できる見た目にする
- 1366px幅のPC利用を主対象にしつつ、狭幅でも破綻しない

---

## 6. セキュリティ・プライバシー基準

- 入力内容を外部APIへ送信しない
- v0.1では永続保存しない
- プロジェクト名はHTMLエスケープして表示する
- 動的HTMLにinline scriptを混ぜない
- 機密情報、認証情報、個人情報を入力しないようUIとREADMEに明記する
- secret scan対象外になるような機密値をサンプルに入れない
