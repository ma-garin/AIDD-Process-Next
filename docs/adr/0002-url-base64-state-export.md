# ADR-0002: 診断データの受け渡しをURLBase64エンコードで実現する

- **Status**: Accepted
- **Date**: 2026-06-16

## Context

コンサルタントモデルでは「クライアントが事前に50問を回答 → コンサルタントがセッションで結果を解釈」するフローが必要。しかしフェーズCはバックエンドなしの静的HTMLサイトであり、サーバーサイドでの状態保存ができない。

代替案として以下を検討した:
1. MarkdownレポートをコピーしてSlack/メールで送る（情報断片化、再現不可）
2. JSONファイルをエクスポート/インポートする（非技術者には敷居が高い）
3. 回答状態をBase64エンコードしてURLに乗せる（バックエンド不要）

## Decision

回答状態をBase64エンコードしてURLクエリパラメータに乗せ、URLを共有することでコンサルタントとクライアントの間でデータを受け渡す。

## Consequences

**メリット:**
- バックエンド不要のためフェーズCのまま実装できる
- URLを送るだけで完結し、操作が直感的
- コンサルタントが同じ画面状態を再現できる

**デメリット・トレードオフ:**
- URLが長くなる（50問分の回答をエンコードするため）
- URLを知っている人なら誰でも診断結果にアクセスできる（機密性なし）
- フェーズAでバックエンドを追加する際に、URL共有とサーバー保存の2系統が生まれる可能性がある
