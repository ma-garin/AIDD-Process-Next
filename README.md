## 何をするツールか

AIを使った開発プロセスが、属人的・場当たり的になっていないかを10領域・50問で自己診断します。

主な出力は以下です。

| 出力 | 内容 |
|---|---|
| 成熟度スコア | Lv.0 未統制 → Lv.3 最適化 |
| 優先リスク | スコア0〜1の項目を重大・高で分類 |
| 改善Issue案 | すぐIssue化できる改善タスク案 |
| Markdownレポート | 診断結果をそのまま共有できる形式で出力 |
| AI自己点検プロンプト | 実装前・PR前・マージ前にAIエージェントへ渡す確認プロンプト |

---

## 主な機能

- **回答**: キーボード操作（0–3で回答 / N で対象外 / ←→で移動）、設問ごとの根拠メモ、BARS基準の常時表示
- **中断/再開**: 進捗を localStorage に自動保存し、次回「前回の続きから再開」
- **見直し**: 全50回答を一覧で俯瞰し、その場で修正できる見直し画面
- **結果**: レーダーチャート（クリックで領域へジャンプ）、回答率による信頼度・暫定表示、次レベルまでの逆算、回答の整合性チェック、前回との差分、結果の根拠提示
- **書き出し**: Markdown / 診断JSON（再取込可）/ スコアCSV / Issue CSV・一括コピー / 共有リンク（URLに回答を埋め込み）/ 成熟度バッジ
- **自己点検**: 実装前・PR前・マージ前 × 対象エージェント別（Claude Code / Codex / Cursor）
- **表示**: ダークモード（OS設定追従 + 手動切替）、印刷/PDF、オフライン動作（PWA）

## 使い方

### Option A — Node.jsで起動

```bash
npm run dev
# http://localhost:3000 を開く
```

初回は `npx serve` がダウンロードされます。

### Option B — Pythonで起動

```bash
cd docs
python3 -m http.server 3000
# http://localhost:3000 を開く
```

### Option C — 直接開く

`docs/index.html` をブラウザで開いても利用できます。
外部CDNは使っていません。レーダーチャートはインラインSVGで描画します。

### 開発・検証コマンド

```bash
npm run check      # 主要JSの構文チェック（node --check）
npm test           # スコアリングの単体テスト（node:test・ゼロ依存）
npm run scan       # 出荷UIへの専有名称混入チェック
npm run done-gate  # 必須成果物の存在ゲート
```

CI（`.github/workflows/ci.yml`）で構文・テスト・禁止語スキャンを実行します。アクセシビリティ／パフォーマンスは Lighthouse（`quality.yml`）で可視化します。

## スコアリング

| スコア | レベル | 状態 |
|---:|---|---|
| 75〜100 | Lv.3 最適化 | 指標・証跡・自己監査に基づき継続改善されている |
| 50〜74 | Lv.2 仕組み化 | ルール、品質ゲート、追跡性が定義されている |
| 25〜49 | Lv.1 管理開始 | 基本ルールはあるが属人的・不安定 |
| 0〜24 | Lv.0 未統制 | AI利用が場当たり的でリスクが高い |

- 各設問は0〜3点
- カテゴリスコアは回答済み項目の（設問重み付き）平均を100点換算。設問重みは暫定で均一
- 対象外（N/A）は母数から除外
- v0.1ではカテゴリ重みを均等10%として扱う
- 回答率が低い場合は総合レベルを「暫定」表示し、信頼度・安定性を併記
- カテゴリ重み・設問重みの値はリサーチゲート完了後に根拠化（現状は暫定）

## セキュリティ・プライバシー

- 入力内容は外部送信されません（計測・クッキーなし。詳細は `PRIVACY.md`）
- データはブラウザ内で処理され、進捗はこの端末の localStorage にのみ保存します
- プロジェクト名欄や根拠メモに秘密情報、認証情報、個人情報を入れないでください
- 動的HTMLは `escapeHtml` でエスケープします
- 動的要素のイベントはdata属性とイベント委譲で扱い、動的HTML内のinline scriptを避けています

## 外部開発者向け注意

作業前に `agent.md` と `sqec.md` を読んでください。

- 設問は `docs/data/questions.js` に集約し、UIへ直書きしない
- スコアリングは `docs/js/scoring.js` に閉じる
- 明示指示なしにバックエンド、認証、DB、AI APIを追加しない
- 生成AIサービス風の派手なデザインではなく、日本のB2B業務ツールとして使える落ち着いたUIを維持する
- `.git`、`.DS_Store`、`.playwright-mcp/`、`__MACOSX/` はコミットしない
