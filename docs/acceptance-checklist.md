# acceptance-checklist — 受け入れチェックリスト

本リストは `sqec.md` の品質ゲート（QG-1〜QG-10）とテスト観点（TC-001〜TC-011）に対応する、機械可読寄りの受け入れ確認項目です。各項目に対応IDを併記します。リリース判定・PRレビュー時に使用します。

---

## 品質ゲート（QG）

- [ ] [QG-1] 主要画面・設問・レポート・自己点検プロンプトが日本語で表示される
- [ ] [QG-2] 低彩度・高可読性で、日本のB2B業務ツールとして違和感のないデザインである
- [ ] [QG-3] 設問は `docs/data/questions.js` に集約され、UIへ直書きされていない
- [ ] [QG-4] 同じ回答から同じスコア・成熟度・リスクが再現される
- [ ] [QG-5] スコア0〜1の項目が優先リスクとして表示される
- [ ] [QG-6] 優先リスクから改善Issue案が生成される
- [ ] [QG-7] Markdownレポートを表示・コピーできる
- [ ] [QG-8] 実装前・PR前・マージ前のAI自己点検プロンプトを表示・コピーできる
- [ ] [QG-9] 入力値がHTMLエスケープされ、外部送信されない
- [ ] [QG-10] `.git`・`.DS_Store`・`.playwright-mcp/`・`__MACOSX/` を含まない

---

## テスト観点（TC）

- [ ] [TC-001] 初期表示: トップ画面が日本語で表示される
- [ ] [TC-002] 診断開始: 1カテゴリ目の1問目が表示される
- [ ] [TC-003] 回答進捗: 回答後に進捗が更新される
- [ ] [TC-004] N/A: 対象外選択時に母数から除外される
- [ ] [TC-005] サンプル: サンプル読込後に結果を表示できる
- [ ] [TC-006] 重大リスク: スコア0の項目が重大リスクとして出る
- [ ] [TC-007] 高リスク: スコア1の項目が高リスクとして出る
- [ ] [TC-008] レポート: Markdownレポートが生成される
- [ ] [TC-009] コピー: レポートと自己点検プロンプトをコピーできる
- [ ] [TC-010] XSS: プロジェクト名へHTMLを入れてもエスケープされる
- [ ] [TC-011] 構文: 主要JSファイルが `node --check` を通る

---

## Definition of Done

以下をすべて満たした場合に「完了」とみなします。

- [ ] 変更範囲が依頼内容に収まっている
- [ ] 関連する QG / TC 項目を確認済みである
- [ ] `node --check docs/js/app.js` が成功する
- [ ] `node --check docs/js/scoring.js` が成功する
- [ ] `node --check docs/data/questions.js` が成功する
- [ ] テストが存在する場合 `node --test` が成功する
- [ ] `scripts/done-gate.sh` が成功する（必須成果物が揃っている）
- [ ] 設問変更がある場合、`docs/research/evidence-matrix.csv` の根拠に紐付け、「設問ID → 根拠ソースID」を記録済みである
- [ ] 外部送信・認証・DB・AI API・外部依存を追加していない
- [ ] 秘密情報・個人情報・認証情報を追加していない
- [ ] 専有文言（TPI NEXT・QA4AIDD 等）を流用していない
- [ ] ユーザー影響のある変更を `CHANGELOG.md` の `[Unreleased]` に追記済みである
